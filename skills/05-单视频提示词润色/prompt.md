将剧情描述按固定字段结构转化为可直接用于 AI 视频生成的结构化提示词。

## 完整 Skill

```markdown
> 使用说明：将需要润色的剧情作为输入，按以下规则生成每个镜头的提示词。返回纯文本提示词，按镜头顺序排列。

## 1. 资产声明（可选）
若润色前存在资产索引，在文首单独列出每一个涉及的资产，按单行格式声明，保留资产标签格式。例如：
<<ref>> 是 xxx

## 2. 镜头划分
根据原剧情划分为多个连续镜头。每个镜头需包含完整的"镜头提示词结构"。

## 3. 输出格式（固定字段与顺序）
严格采用下列【镜头提示词结构】重组并输出每个镜头提示词，返回纯文本提示词。字段顺序严格为：

镜头1，【时长】x秒  
【主体】  
【场景】  
【核心动作】  
【台词】  
【镜头语言】  
【光影/滤镜】  
【动态场景/环境特效】

## 4. 字段内容规则

- 【主体】（必填）  
  格式："<角色名> + 动作、表情、道具交互"。将同一角色的动作、表情与道具交互合并为一完整长句，用逗号分隔，禁止使用代词或模糊称谓。若有陪体人物，紧接写为 ", <陪体角色名> + 动作/表情"。
  示例：张强双手发抖，眉头紧锁，右手死死拽着破旧的皮夹。

- 【场景】  
  标明拍摄地点与时间（如"夜晚的巷道"），可补充关键环境细节。须与原文语义一致。

- 【核心动作】  
  用高度视觉化语言替代平淡动作，突出瞬间张力或情绪反差。尽量一句或两句完成。

- 【台词】（必填）  
  必须逐字填入该镜头所有台词或独白，格式：张强(低声): "我没别的选择。"  
  绝对禁止改写、删减或翻译原文台词；若无台词写"无"。

- 【镜头语言】（必填）  
  写明具体拍摄角度（平视/仰拍/俯拍/越过<角色名>肩膀平视/POV等）及节奏（慢推、快切、闪切）。禁止含糊描述。

- 【光影/滤镜】  
  描述光照方向、色调、质感处理或特定滤镜建议，以传达情绪基调。

- 【动态场景/环境特效】  
  说明物理动态元素或小型特效（飘动窗帘、破碎玻璃、雨滴、尘埃等）。可标注音效并注明 J-cut 或 L-cut；若无写"无"。

## 5. 风格与使用约束
- 描述应清晰、可执行、面向影像实现；用词中性，避免过度文学化
- 严格保留原文台词与人物信息，不得增减或改写
- 每字段以一句或两句为宜，尽量不超过三行

## 6. 结束语说明
在所有镜头提示词后，必须增加以下补充说明（逐字保留）：  
"严格按照分镜脚本进行生成，参考上传图的人物形象、场景和道具。不要生成字幕。"
```
